# 代付管理系统（已转管理后台）开发任务记录

更新时间：2026-03-02

## 1. 项目现状概览
- 当前项目已从“代付链接系统”重构为“用户/店铺/商品/购物车/统计”后台。
- 前端已移除 Mock，全部走后端接口。
- 已新增微信内 JSAPI 支付主链路（购物车多选结算 -> 下单 -> 回调更新）。

## 2. 已完成任务
1. 后端业务重构
- 新模块：用户、店铺、商品、购物车、统计。
- 统一返回：`Result<T>` + `PageResult<T>`。
- 删除旧代付订单模块（`pay_order/pay_notify_log` 旧逻辑）。

2. 数据库重构
- 新核心表：`sys_user`、`store`、`product`、`cart_item`、`user_stats_daily`。
- 支付新增表：`payment_order`、`payment_order_item`、`payment_notify_log`。
- 迁移文件：
`backend/src/main/resources/db/migration/V1__init.sql`
`backend/src/main/resources/db/migration/V2__payment.sql`

3. 前端重构与优化
- 页面模块：用户管理、店铺管理、商品管理、购物车、数据统计。
- 页面结构优化：标题区 + 筛选卡 + 表格卡 + 分页。
- 增加 `v-loading`、空态、危险操作二次确认。
- 购物车页面支持多选后发起支付。

4. 微信支付（JSAPI）第一版
- 后端接口：
`POST /api/pay/checkout`
`POST /api/pay/wechat/jsapi`
`GET /api/pay/orders/{orderNo}`
`POST /api/pay/wechat/notify`
- 流程：
创建结算单 -> 微信统一下单 -> 前端唤起 `WeixinJSBridge` -> 回调验签解密 -> 标记支付成功 -> 扣库存 -> 删除已支付购物车项。

5. 支付前端第二轮更新（已完成）
- 购物车页新增“最近支付订单”信息面板（订单号、金额、状态、prepayId）。
- 新增“查单”按钮，调用 `GET /api/pay/orders/{orderNo}` 手动刷新支付状态。
- 微信支付前增加二次确认弹窗（显示选中条数和支付金额）。
- 新增按用户缓存 OpenID（`localStorage`），切换用户自动回填。
- 增强支付错误提示与状态轮询后的反馈文案。

## 4. 必填配置（微信支付）
在 `backend/src/main/resources/application.yml` 填写：
- `app.wechat.app-id`
- `app.wechat.mch-id`
- `app.wechat.mch-serial-no`
- `app.wechat.private-key-pem`
- `app.wechat.platform-public-key-pem`
- `app.wechat.api-v3-key`
- `app.wechat.notify-url`
- `app.wechat.gateway`（默认官方网关）
- `app.wechat.order-expire-minutes`

## 6. 已知问题与风险
1. 前端部分页面中文存在编码异常（出现乱码），不影响接口调用但影响显示，需要统一 UTF-8 修复。
2. 微信支付回调已做验签与解密，但缺少更完整的异常补偿策略（如回调重试队列）。
3. 当前无退款、关单、支付超时批处理任务。
4. 未做服务端 JWT 鉴权拦截（`SecurityConfiguration` 仍放开 `/api/**`）。
5. 支付功能缺少自动化测试（单元测试/集成测试未补齐）。
